# Use Cases

# UC-001 Регистрация пользователя

## Актор
Покупатель

## Предусловие
Пользователь не зарегистрирован

## Основной сценарий
1. Пользователь открывает страницу регистрации
2. Вводит email
3. Вводит пароль
4. Нажимает кнопку «Зарегистрироваться»
5. Система проверяет данные
6. Создается учетная запись

## Альтернативный сценарий
Email уже зарегистрирован

# UC-002 Авторизация

## Актор
Покупатель

## Основной сценарий
1. Пользователь открывает страницу входа
2. Вводит логин и пароль
3. Нажимает кнопку входа
4. Система выполняет проверку
5. Пользователь входит в систему

# UC-003 Поиск товара

## Актор
Покупатель

## Основной сценарий
1. Пользователь вводит название товара
2. Система выполняет поиск
3. Отображается список товаров

# UC-004 Оформление заказа

## Актор
Покупатель

## Основной сценарий
1. Пользователь добавляет товар в корзину
2. Переходит к оформлению
3. Вводит адрес доставки
4. Выбирает способ оплаты
5. Подтверждает заказ
6. Заказ создается

# UC-005 Оплата заказа

## Актор
Покупатель

## Основной сценарий
1. Пользователь выбирает оплату
2. Вводит данные карты
3. Подтверждает оплату
4. Система получает подтверждение платежа
5. Заказ получает статус «Оплачен»
